1

Scale the prototype-to-production pattern

The UI POD proved the workflow is viable. The focus now is documenting the pattern and expanding it to more feature teams.

2

Unblock external testing

Finalise the security review for public GitHub Pages so prototypes can be used in real external user testing sessions — a major unlock for the research workflow.

3

Define metrics baseline

Establish the metrics we want to track going forward: time-to-prototype, handoff quality, and team confidence scores.

4

Promote builder culture

The research hub shows value beyond product teams. Identify two or three more disciplines where AI prototyping could enhance workflows.
