From drawing pixels in Figma to prompting in Claude

Two months ago, our prototyping workflow was manual, slow, and disconnected from engineering. Today, AI is changing every step — from the first idea to a working interface.

↩ Two months ago
"Hey Designer, can you mock up a few directions I can review by the end of the week?"
  • ✕ Open Figma, build frames from scratch or find a template to adapt
  • ✕ Manually place every component, set every spacing value
  • ✕ Share a static link — stakeholders see pixels, not behaviour
  • ✕ Iterate by redrawing; each round takes hours
  • ✕ Hand off a spec doc; engineer re-interprets the design
→ Today
"Hey team, let's get together and visualize a few directions on a call today"
  • ✓ Prompt Claude Code with the brief, utilizing design system
  • ✓ Get a working, interactive code prototype in minutes
  • ✓ Share a live URL — stakeholders click through real interactions
  • ✓ Iterate by prompting; rounds take minutes, not hours
  • ✓ The prototype is a real artefact — testable, iterable, and engineer-readable

Three significant changes

The nature of our work has fundamentally shifted

Design is interactive from day one
Instead of creating static images from written requirements teams are creating interactive prototypes, together, in real time.
Proven
Design and engineering are converging
Prototypes are now real, working code — which means the gap between "designed" and "built" is narrowing fast.
Proven
Our roles are evolving
Designers and Product managers are writing and prompting real code — not as a novelty, but as a core part of how they work.
Expanding

What we're doing to scale this process

The foundation is there, now we need to turn patterns into practice.

1
Secure the prototype
Establish clear guidelines for what can be shared externally, how prototypes are hosted, and ensuring sensitive data never surfaces in a testing environment.
2
Document the prototype-to-production process
Codify the handoff checklist, quality standards, and expectations for when a prototype is production-ready versus still exploratory — so any team can replicate it.
3
Establish metrics baselines
Define what we're tracking and set a baseline: time from brief to prototype, handoff quality, and team confidence. Without it, we can't demonstrate value over time.
4
Enable more teams to work this way
Scale the practice beyond the early adopters. Share what's working, reduce the barrier to entry, and give more teams the tools and confidence to prototype with AI as a default — not an exception.
